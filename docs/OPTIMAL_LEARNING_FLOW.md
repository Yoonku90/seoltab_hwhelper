# 최적의 학습 플로우 설계

## 학습 과학 원리 기반 설계

### 핵심 원리
1. **Spaced Repetition (간격 반복)**: 시간이 지날수록 복습 간격을 늘림
2. **Active Recall (능동적 회상)**: 단순 읽기가 아닌 질문-답변 형식
3. **Interleaving (교차 학습)**: 여러 개념을 섞어서 학습
4. **Elaborative Encoding (정교화 인코딩)**: 의미 있는 연결과 설명
5. **Testing Effect (테스트 효과)**: 테스트가 학습을 강화

## 학습 플로우 설계

### Phase 1: 수업 직후 (0-1시간) - "따끈따끈 요약본"
**목표**: 수업 내용을 즉시 정리하고 핵심을 파악

1. **요약본 생성** (`/admin/lecture-summary`)
   - Room ID 입력
   - STT + 교재 이미지 결합
   - 유은서 쌤 스타일 요약본 생성
   - 핵심 개념, 놓친 부분, 오늘의 미션 포함

2. **즉시 복습** (`/admin/lecture-summary?reviewProgramId=...`)
   - 요약본을 보면서 튜터와 대화
   - 능동적 회상: "이 개념이 뭐였지?" 질문
   - 놓친 부분 집중 복습

3. **학습 완료 및 저장**
   - "✅ 학습 완료 및 저장" 버튼
   - 학생 ID에 연결
   - 다음 단계로 이동

### Phase 2: 숙제 시간 (1-3시간 후) - "은서쌤과 숙제하자"
**목표**: 배운 내용을 문제에 적용

1. **숙제 페이지** (`/homework`)
   - "비법 노트 다 봤어? 숙제도 도와줄 수 있는데, 지금 같이 해볼래?"
   - 이미지 업로드 (문제집/시험지)
   - 랑쌤/준쌤과 티키타카

2. **문제 해결 과정**
   - 힌트 제공 (직접 답 주지 않음)
   - 단계별 안내
   - 실수 학습 (틀려도 괜찮아, 왜 틀렸는지 이해)

3. **학습 기록**
   - 풀었던 문제 저장
   - 어려웠던 문제 마킹
   - 다음 복습에 반영

### Phase 3: 다음날 복습 (24시간 후) - "어제 배운 거 기억나?"
**목표**: 망각 곡선 대응, 장기 기억 강화

1. **복습 알림** (추후 구현)
   - "어제 배운 [과목명] 복습할 시간이야!"
   - 요약본 다시 보기
   - 간단한 퀴즈

2. **간격 반복 복습**
   - 1일 후, 3일 후, 7일 후, 14일 후
   - 점점 간격 늘리기
   - 자동으로 복습 프로그램 생성

### Phase 4: 주간/월간 총정리
**목표**: 여러 개념을 연결하고 통합 이해

1. **주간 복습**
   - 한 주간 배운 내용 통합
   - 개념 간 연결
   - 종합 문제

2. **월간 평가**
   - 학습 진도 확인
   - 약점 파악
   - 다음 학습 계획

## 구현 우선순위

### Phase 1 (즉시 구현)
- ✅ 요약본 생성 및 표시
- ✅ 복습 프로그램 (STT + 이미지 + 서머리)
- ✅ 학습 완료 및 저장 → 숙제 페이지 이동
- ✅ 숙제 도와주기

### Phase 2 (다음 구현)
- [ ] 복습 알림 시스템
- [ ] 간격 반복 스케줄링
- [ ] 학습 진도 추적
- [ ] 개인화된 복습 추천

### Phase 3 (장기 구현)
- [ ] 주간/월간 총정리
- [ ] 학습 분석 대시보드
- [ ] 부모/선생님 리포트

## 사용자 여정 (User Journey)

```
1. 수업 종료
   ↓
2. Room ID로 요약본 생성
   ↓
3. 요약본 읽기 (10분)
   - 핵심 개념 파악
   - 놓친 부분 확인
   ↓
4. 복습 프로그램 시작
   - 튜터와 대화하며 복습
   - 능동적 회상 연습
   ↓
5. 학습 완료 및 저장
   ↓
6. 숙제 페이지로 이동
   - "이제 은서쌤과 숙제하자!"
   ↓
7. 숙제 이미지 업로드
   ↓
8. 랑쌤/준쌤과 문제 풀기
   - 힌트 받기
   - 단계별 학습
   ↓
9. 숙제 완료
   ↓
10. (다음날) 복습 알림
    - 어제 배운 내용 다시 보기
    - 간단한 퀴즈
```

## 학습 효과 극대화 전략

### 1. 즉시 피드백
- 문제를 풀면 바로 정답 확인
- 틀렸을 때 왜 틀렸는지 설명
- 다음에 어떻게 접근할지 제안

### 2. 점진적 난이도
- 쉬운 문제부터 시작
- 점점 어려운 문제로
- 학생 수준에 맞춘 난이도 조절

### 3. 성취감 부여
- "오늘 5개 문제 풀었어!" 같은 진행 상황 표시
- "이 개념 완벽하게 이해했어!" 같은 격려
- 학습 스트릭 (연속 학습 일수)

### 4. 개인화
- 학생의 약점 파악
- 집중 복습 영역 추천
- 학습 스타일에 맞춘 설명

## 기술적 구현 방향

### 데이터 구조
```typescript
{
  studentId: string,
  learningSessions: [
    {
      roomId: string,
      summaryId: string,
      completedAt: Date,
      reviewSchedule: [
        { date: Date, completed: boolean }, // 1일 후
        { date: Date, completed: boolean }, // 3일 후
        { date: Date, completed: boolean }, // 7일 후
      ]
    }
  ],
  homeworkSessions: [
    {
      imageUrl: string,
      problems: [
        { problemNumber: number, status: 'solved' | 'stuck' | 'question' }
      ],
      completedAt: Date
    }
  ]
}
```

### API 엔드포인트
- `POST /api/learning/complete` - 학습 완료 기록
- `GET /api/learning/review-schedule` - 복습 일정 조회
- `POST /api/learning/review` - 복습 완료 기록
- `GET /api/learning/progress` - 학습 진도 조회
