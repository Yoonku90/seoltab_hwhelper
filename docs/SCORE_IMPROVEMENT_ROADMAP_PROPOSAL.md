# 점수 향상 로드맵 기능 제안

## 🎯 사용자 요청

### 핵심 아이디어:
- 학생들이 학습 활동과 실제 점수 향상의 연관성을 시각적으로 확인
- "이것만 달성하면 점수 10점 무조건 올라!" 같은 구체적인 목표 제시
- "숙제를 한달 꾸준히 하면 점수 10점 올라" 같은 통계 기반 로드맵

### 예시:
- 영어: 영단어만 외우는 것만으로도 성적 10점 상승
- 수학: 문제 100개 풀면 점수 10점 상승
- 국어: 문법 개념 5개 마스터하면 점수 10점 상승

## 📊 데이터 수집 방법

### 방법 1: 실제 성적 데이터 수집 (가장 정확) ⭐

#### 데이터 소스:
1. **학생이 직접 입력**
   - 모의고사/기출 점수 입력
   - 시험 점수 입력
   - 주기적으로 업데이트 (월 1회, 시험 전후 등)

2. **학원/학교 연동** (향후)
   - 학원 관리 시스템과 연동
   - 학교 성적표 자동 수집

#### 데이터 구조:
```typescript
interface ScoreRecord {
  _id?: string;
  studentId: string;
  subject: 'korean' | 'english' | 'math' | 'social' | 'science';
  score: number; // 점수 (0-100)
  maxScore: number; // 만점
  testType: 'mock' | 'school' | 'practice' | 'homework';
  testDate: Date;
  testName?: string; // "2024년 3월 모의고사", "중간고사" 등
  metadata?: {
    grade?: string; // 학년
    semester?: string; // 학기
  };
  createdAt: Date;
}

interface LearningActivity {
  _id?: string;
  studentId: string;
  subject: string;
  activityType: 
    | 'vocabulary_study' // 영단어 학습
    | 'problem_solve' // 문제 풀이
    | 'concept_review' // 개념 복습
    | 'homework_complete' // 숙제 완료
    | 'review_program' // 복습 프로그램
    | 'quiz_complete'; // 퀴즈 완료
  activityData: {
    count?: number; // 영단어 개수, 문제 개수 등
    duration?: number; // 학습 시간 (분)
    accuracy?: number; // 정답률
    topics?: string[]; // 학습한 주제들
  };
  date: Date;
  createdAt: Date;
}
```

### 방법 2: 학습 활동 데이터 분석 (현재 시스템 활용)

#### 현재 수집 가능한 데이터:
1. **복습 프로그램 완료 횟수**
   - `review_programs` 컬렉션에서 완료된 프로그램 수
   - 주제별, 과목별 통계

2. **문제 풀이 통계**
   - `problems` 컬렉션에서 해결한 문제 수
   - 정답률, 소요 시간

3. **학습 이벤트 로그**
   - `learning_events` 컬렉션에서 학습 패턴 분석
   - 세션 시간, 개념 학습 횟수

4. **퀴즈/연습 통계**
   - 정답률, 완료한 퀴즈 수

#### 데이터 분석:
```typescript
interface LearningAnalytics {
  studentId: string;
  subject: string;
  period: {
    start: Date;
    end: Date;
  };
  metrics: {
    // 학습 활동
    reviewProgramsCompleted: number;
    problemsSolved: number;
    vocabularyWordsStudied: number; // 영단어 개수
    conceptsLearned: number;
    totalStudyTime: number; // 총 학습 시간 (분)
    averageAccuracy: number; // 평균 정답률
    
    // 점수 변화
    scoreChange?: number; // 기간 동안 점수 변화
    initialScore?: number; // 기간 시작 시 점수
    finalScore?: number; // 기간 종료 시 점수
  };
}
```

## 🔍 통계 분석 방법

### 방법 1: 코호트 분석 (Cohort Analysis)

#### 개념 (쉽게 설명):
**코호트 분석**은 비슷한 특성을 가진 학생들을 그룹으로 묶어서 분석하는 방법이에요.

예를 들어:
- "영단어 1000개 외운 학생들"을 한 그룹으로 묶기
- 이 그룹의 점수 변화를 추적
- 평균적으로 얼마나 점수가 올랐는지 계산

#### 실제 예시:
```
학생 A: 영단어 1000개 외움 → 점수 60점 → 70점 (+10점)
학생 B: 영단어 1000개 외움 → 점수 65점 → 78점 (+13점)
학생 C: 영단어 1000개 외움 → 점수 55점 → 66점 (+11점)

→ 이 3명의 평균 점수 상승: (10 + 13 + 11) / 3 = +11.3점

결과: "영단어 1000개 외운 학생들의 평균 점수 상승: +11.3점"
```

#### 구현:
```typescript
// 예: 영단어 1000개 외운 학생 그룹
const cohort = students.filter(s => 
  s.learningActivities.vocabularyWordsStudied >= 1000
);

// 이 그룹의 평균 점수 변화
const avgScoreChange = cohort.reduce((sum, s) => 
  sum + (s.finalScore - s.initialScore), 0
) / cohort.length;

// 결과: "영단어 1000개 외운 학생들의 평균 점수 상승: +12.5점"
```

#### 왜 유용한가?
- 개별 학생의 변수(능력, 환경 등)를 제외하고, **학습 활동 자체의 효과**를 측정
- "이 활동을 하면 평균적으로 이만큼 점수가 오른다"는 통계적 근거 제공

## 🔧 구현 우선순위

### ⚠️ 데이터가 0인 상태에서도 시작 가능한 방법

#### 방법 1: 교육 연구 데이터 활용 (즉시 가능) ⭐
- 교육부, 교육연구원 등에서 발표한 통계 데이터 활용
- 예: "영단어 학습과 영어 점수 상승의 상관관계 연구"
- 학술 논문, 교육 통계 자료 참고

#### 방법 2: 업계 벤치마크 데이터 활용
- 다른 학습 플랫폼의 공개된 통계 데이터
- 교육 업계의 일반적인 경험치 활용

#### 방법 3: 추정치로 시작 (Cold Start)
- 교육 전문가의 경험 기반 추정
- 예: "일반적으로 영단어 1000개 외우면 점수 10점 상승"
- 명시적으로 "추정치"라고 표시

#### 방법 4: 점수 입력 기능 먼저 구현
- 학생들이 점수를 입력하도록 유도
- 데이터가 쌓이면 실제 통계로 업데이트
- 초기에는 추정치 사용, 나중에 실제 데이터로 교체

### Priority 1: 점수 입력 기능 + 추정치 로드맵 (즉시 가능) ⭐
- 학생이 점수를 직접 입력할 수 있는 UI
- 점수 기록 저장 및 조회
- **추정치 기반 로드맵 제공** (교육 연구 데이터 또는 전문가 추정)
- 명시: "통계 기반 추정치 (실제 데이터 축적 중)"

### Priority 2: 기본 통계 (데이터 축적 후)
- 학습 활동과 점수 변화의 기본 통계
- 간단한 상관관계 분석
- 추정치를 실제 통계로 점진적 교체

### Priority 3: 로드맵 UI (통계 데이터 준비 후)
- 점수 향상 로드맵 화면
- 목표 설정 및 진행도 추적
- 실제 통계 데이터 기반 업데이트

### Priority 4: 고급 분석 (충분한 데이터 후)
- 머신러닝 예측 모델
- 개인화된 추천
- 코호트 분석, 상관관계 분석
